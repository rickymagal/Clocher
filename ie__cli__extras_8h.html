Detailed Description

Small, self-contained CLI extras shared across benchmark-style entrypoints.

This header intentionally defines only the "extras" flags that are common across harness entrypoints (batching, prompts file, rounds, and optional expected-token verification). It does not impose a particular argument parser implementation.

Expected tokens file format (one entry per line): <prompt_id><whitespace><token0,token1,token2,...>

  • Lines starting with '#' and empty lines are ignored.
  • prompt_id can be decimal or 0x-prefixed hexadecimal.
  • prompt_id is a stable 64-bit FNV-1a hash of the prompt bytes (UTF-8).

The intent is to allow a strict harness to verify that an engine build is stable (token IDs are identical) across runs and across environments.
