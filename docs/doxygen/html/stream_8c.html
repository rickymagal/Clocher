Best-effort cache discovery on Linux sysfs.

Vendor sysfs layouts vary; we read a small fixed set of potential index paths and rank the discovered sizes. The mapping "largest => L3, second => L2, third => L1" is a rough approximation but works reasonably well for tuning thresholds without hard dependencies.

If nothing is found, outputs remain zero.

Parameters
l1Out: L1D size in bytes (best-effort).
l2Out: L2 size in bytes (best-effort).
l3Out: L3/LLC size in bytes (best-effort).
