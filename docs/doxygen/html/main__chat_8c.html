Detailed Description

Interactive CLI for tokenizer validation and (later) text generation.

This binary is meant to be "human-facing":

  • encode: text -> ids
  • decode: ids -> text
  • roundtrip: encode + decode

Once you wire real generation into ie_api.c / ie_engine_generate(), you can add a "prompt -> generate -> decode" mode here without touching the benchmark harness behavior in main_infer.c.

Build integration:

  • Add this file to your Makefile as a separate target (e.g. chat).
  • Link with the same engine objects you already build, plus tokenizer_gptoss.c.
