Detailed Description

In-memory cache for reconstructed (materialized) tensors.

This cache exists to convert the dedup storage format (defaults + mask + exceptions) into a stable, directly usable byte buffer.

Key requirements for real hits:

  • Cache keys must be stable (e.g. tensor name or tensor index).
  • The engine must request the same key repeatedly across tokens/steps.
  • Cached pointers must remain valid for the duration of the run.

This is a generic cache: it does not know about NUMA replication; callers can replicate "hot" cached blobs separately when needed.
