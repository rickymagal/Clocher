Detailed Description

Header-compliant, CI-friendly implementation of the public engine API.

Design goals

  • Strict compatibility with ie_api.h (types, signatures, status codes).
  • Deterministic behavior: pseudo-random token IDs derived from a per-engine seed and the input prompt hash — stable across runs for identical inputs.
  • CI-friendly: never depends on external model files here; the “real model required” gating is performed by the CLI layer (main_infer.c) via IE_REQUIRE_MODEL.
  • Safe metrics: always returns a valid ie_metrics_t snapshot; callers (CLI/tests) compute wall-time/TPS and may override fields as needed.

Implementations may output fewer tokens (e.g., due to EOS). The caller owns the out_tokens buffer and must supply capacity for max_new_tokens.

Parameters

[in]promptNUL-terminated prompt text.

[out]out_tokensOutput buffer for token IDs (size >= max_new_tokens).

Return values
IE_OKon success.
non-zeroon failure.

Generate up to max_new_tokens from prompt.

Write contract
  • If max_new_tokens == 0, nothing is written to out_tokens and *out_count is set to 0; returns IE_OK.
  • On success, *out_count equals the number of tokens produced (≤ max).
  • Output is deterministic for identical (engine seed, prompt).
