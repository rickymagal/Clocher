Detailed Description

Minimal public API implementation (engine create/destroy/generate).

This is the stable entrypoint used by the CLI and benchmark harness.

The core inference kernels and model loading logic may evolve independently, but the exported signatures in ie_api.h should remain stable.

Current behavior:

  • ie_engine_create stores device/model_dir and copies params (best-effort).
  • ie_engine_generate produces deterministic token ids (prompt-dependent).

The benchmark harness measures "strict touch" work in main_infer.c; generation here is intentionally lightweight and deterministic.
