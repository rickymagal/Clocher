Detailed Description

Loader for block-sparse weight matrices from a compact binary format.

This translation unit implements a minimal on-disk format and corresponding loader for ie_block_sparse_matrix_t. It is designed to be:

  • Compact: only non-zero blocks and essential metadata are stored.
  • Portable across typical x86_64 setups: assumes little-endian layout and 32-bit integers for sizes and indices.
  • Strict but forgiving: basic validation is performed on all fields (magic, version, dimensions, counts). On failure, no partially loaded state is left in the descriptor.

The on-disk layout is:

struct ie_block_sparse_header_disk {
char magic[4]; // "IEBS"
uint32_t version; // currently 1
uint32_t rows; // total dense rows
uint32_t cols; // total dense columns
uint32_t block_rows; // block height
uint32_t block_cols; // block width
uint32_t nnzb; // number of non-zero blocks
uint32_t reserved[4]; // reserved for future use (zeroed)
};
// Followed by:
uint32_t row_ptr[n_block_rows + 1];
uint32_t col_idx[nnzb];
float values[nnzb * block_rows * block_cols];

where:

  • n_block_rows = ceil(rows / block_rows)
  • n_block_cols = ceil(cols / block_cols)

The loader validates that:

  • block_rows and block_cols are non-zero.
  • rows and cols are non-zero.
  • nnzb is consistent with the size of the values array.

Error handling:

  • Functions return ie_sparse_status_t codes.
  • On any error, the output descriptor is left in a zeroed state.

ie_block_sparse_matrix_t *  out 
)

Load a block-sparse matrix from a binary file.

This function reads the on-disk format described in the file header documentation above and populates out with a fully initialized ie_block_sparse_matrix_t instance.

On success:

  • ie_block_sparse_matrix_t::rows and cols reflect the dense dimensions.
  • ie_block_sparse_matrix_t::block_rows and block_cols reflect the block layout.
  • ie_block_sparse_matrix_t::n_block_rows and n_block_cols are derived from the dense dimensions and block sizes.
  • ie_block_sparse_matrix_t::row_ptr, col_idx, and values are allocated and filled.

On failure:

  • IE_SPARSE_ERR_* is returned.
  • out is left in a zeroed / empty state (safe to reuse or release).
Parameters
pathFilesystem path to the binary block-sparse matrix.
outDestination descriptor (must not be NULL).
Returns
IE_SPARSE_OK on success, error code otherwise.

void ie_block_sparse_release ( ie_block_sparse_matrix_t *  m)

Release all heap-allocated storage owned by a block-sparse matrix.

If m is NULL, this function is a no-op. Otherwise it:

  • Calls free on ie_block_sparse_matrix_t::row_ptr, ie_block_sparse_matrix_t::col_idx, and ie_block_sparse_matrix_t::values when non-NULL.
  • Resets all scalar fields to zero.
  • Sets all pointer fields to NULL.

After this call, the descriptor is in the same state as after ie_block_sparse_init and may be reused.
