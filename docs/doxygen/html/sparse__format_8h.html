Detailed Description

In-memory representation and loader interface for block-sparse matrices.

This header defines:

  • The in-memory layout for block-sparse matrices used by the engine (see ie_block_sparse_matrix_t).
  • A small status-code enum for sparse-related operations (see ie_sparse_status_t).
  • Utility helpers for computing block grid sizes and densities.
  • The public loader function ie_block_sparse_load, which reads a compact binary representation into an in-memory descriptor.
  • A reference single-precision GEMV kernel (see ie_gemv_block_sparse_f32).

The chosen layout is a block-row CSR (BSR):

  • The full matrix has rows x cols elements.
  • It is partitioned into blocks of block_rows x block_cols.
  • Block rows are numbered from 0 to n_block_rows-1, where n_block_rows = ceil(rows / block_rows).
  • Block columns are numbered from 0 to n_block_cols-1, where n_block_cols = ceil(cols / block_cols).
  • Only non-zero blocks are stored; zero blocks are omitted.

Storage:

  • ie_block_sparse_matrix_t::row_ptr has size (n_block_rows + 1). For block-row br, the non-zero block indices are in the range: [ row_ptr[br], row_ptr[br + 1] ).
  • ie_block_sparse_matrix_t::col_idx has size nnzb, where each entry stores a block-column index (0-based).
  • ie_block_sparse_matrix_t::values has size: nnzb * block_rows * block_cols. Values for block j are laid out in row-major order:
    offset = j * (block_rows * block_cols);
    // element (i, k) inside the block:
    values[offset + i * block_cols + k]

Notes:

  • Tail blocks (on the last block-row/column) may extend beyond the actual matrix dimensions. Consumers must clip accesses using the rows and cols fields.
  • The on-disk format is defined by the implementation in engine/src/sparse_io.c and consumed via ie_block_sparse_load.

typedef struct ie_block_sparse_matrix ie_block_sparse_matrix_t

In-memory representation of a block-sparse matrix.

This structure contains all metadata and storage required to perform matrix-vector products in block-sparse form. It is owned by the caller; loader functions such as ie_block_sparse_load will allocate and populate the arrays and can later be released using ie_block_sparse_release.

Lifetime rules:

  • All pointer fields are either NULL (when nnzb is zero) or point to heap-allocated memory obtained via malloc.
  • After a successful load, the caller must eventually call ie_block_sparse_release on the descriptor.
  • After release, the descriptor is left in a zeroed state and may be reused or safely destroyed.

Load a block-sparse matrix from a binary file.

This function reads the on-disk format defined by the implementation in engine/src/sparse_io.c and populates out with a fully initialized ie_block_sparse_matrix_t instance.

On success:

  • ie_block_sparse_matrix_t::rows and cols reflect the dense dimensions.
  • ie_block_sparse_matrix_t::block_rows and block_cols reflect the block layout.
  • ie_block_sparse_matrix_t::n_block_rows and n_block_cols are derived from the dense dimensions and block sizes.
  • ie_block_sparse_matrix_t::row_ptr, col_idx, and values are allocated and filled.

On failure:

  • IE_SPARSE_ERR_* is returned.
  • out is left in a zeroed / empty state (safe to reuse or release).

Perform a single-precision block-sparse matrix-vector product.

This function computes:

y = W * x (+ bias, if provided)

where W is represented in block-sparse form using ie_block_sparse_matrix_t. The input vector x has length m->cols, and the output y has length m->rows.

Tail blocks:

  • If the matrix size is not an exact multiple of the block dimensions, the implementation clips accesses to rows and cols, ensuring no out-of-bounds reads/writes occur.

Threading:

  • This function is intentionally single-threaded. Thread-level parallelism should be managed by higher-level components (e.g., a thread pool).
