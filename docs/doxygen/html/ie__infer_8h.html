Detailed Description

Inference interface for the GPT-OSS runtime.

Performance notes:

  • INT4 runs use Q4_0 weight GEMV kernels (runtime-dispatched to AVX2/FMA when available).
  • Per-32-column scales may be stored as BF16 (2 bytes) or FP8(E4M3) (1 byte) depending on artifacts.

This header defines the stable ABI used by the public API layer (ie_api.c) and by the CLI harness (main_infer.c) to run inference.

The runtime implementation is model-specific and lives under engine/src/runtime. The interface is intentionally minimal:

  • Create/destroy an inference context bound to a device + weights + hparams.
  • Prefill a prompt (multi-token).
  • Step one token at a time.

The KV cache storage is provided by the caller (one cache per layer) via ie_kv_cache so the runtime can update attention state incrementally.
