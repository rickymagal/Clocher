Detailed Description

In-memory representation of a block-sparse matrix.

This structure contains all metadata and storage required to perform matrix-vector products in block-sparse form. It is owned by the caller; loader functions such as ie_block_sparse_load will allocate and populate the arrays and can later be released using ie_block_sparse_release.

Lifetime rules:

  • All pointer fields are either NULL (when nnzb is zero) or point to heap-allocated memory obtained via malloc.
  • After a successful load, the caller must eventually call ie_block_sparse_release on the descriptor.
  • After release, the descriptor is left in a zeroed state and may be reused or safely destroyed.

◆ values

float* ie_block_sparse_matrix::values

Dense values for all non-zero blocks, in row-major order.

Size is nnzb * block_rows * block_cols. For block j, the data is laid out as: values[j * (block_rows * block_cols) + i * block_cols + k] representing the element (i, k) in the local block coordinates.

The current implementation uses 32-bit floats. Additional data types (e.g., BF16) can be added later via separate descriptors or metadata.
