Block-sparse GEMV (FP32): y = W * x (+ optional bias).

This entry point operates on matrices stored in block-row CSR (BSR) layout described by ie_block_sparse_matrix_t. Backends are free to implement native sparse kernels; if a backend does not implement this method, the implementation falls back to a CPU block-sparse kernel when possible.

Parameters
devDevice handle.
mBlock-sparse matrix descriptor (must be fully initialized).
xInput vector of length m->cols.
yOutput vector of length m->rows.
biasOptional bias vector of length m->rows (may be NULL).
Returns
0 on success; non-zero if unimplemented or error.

Block-sparse GEMV (FP32): y = W * x (+ optional bias).
