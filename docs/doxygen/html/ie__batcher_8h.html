Detailed Description

Public API for asynchronous prompt prefetch + tokenization batcher.

The batcher asynchronously tokenizes input prompts on background worker threads and enqueues results in a bounded ring buffer. The consumer reads contiguous microbatches (views) directly from the ring without copying, then advances to release those items.

Ownership:

  • The tokenizer callback must allocate *out_ids for each item.
  • The batcher takes ownership on enqueue and frees payloads on advance.

Create a new asynchronous tokenizer batcher.

Spawns up to n_workers threads that prefetch and tokenize prompts into a bounded ring buffer. The consumer should call ie_batcher_get followed by ie_batcher_advance in a loop until the stream is drained.

Defaults:

  • If microbatch == 0, it is treated as 1.
  • If queue_capacity == 0, it is treated as max(microbatch, 8).
  • If n_workers == 0, it is treated as 1.
Parameters
promptsArray of input UTF-8 prompt strings (not owned; must outlive the batcher).
n_promptsNumber of prompts in prompts.
microbatchMaximum items returned per view. Use 0 for default.
queue_capacityRing capacity in items. Use 0 for default.
n_workersNumber of tokenizer worker threads. Use 0 for default.
tokenize_cbUser tokenizer callback (must not be NULL).
user_ctxOpaque pointer forwarded to tokenize_cb; may be NULL.
Returns
ie_batcher_t* Handle on success, or NULL on allocation/spawn error.

int ie_batcher_get ( ie_batcher_t * b,
ie_batcher_view_t * out_view )

Obtain a contiguous slice of ready items without copying.

If no items are ready but production is still in progress, this function blocks until an item is available (or the batcher is stopping).

When this function returns 0, the stream is drained; out_view->done will be set to 1 and out_view->items will be NULL.

Parameters
bBatcher handle (non-NULL).
out_viewOutput view describing a contiguous span of items.
Returns
int 1 if a non-empty view was produced; 0 if the stream is drained.
