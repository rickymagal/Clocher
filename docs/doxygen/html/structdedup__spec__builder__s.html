Detailed Description

Builder internal representation.

The builder keeps:

  • dynamic array of entries
  • a single growing string table (with a leading '\0' sentinel)

This is intentionally simple: no hash table for string interning (Step 1 scope). The extractor typically interns O(10^2) names, so linear lookup is acceptable.
