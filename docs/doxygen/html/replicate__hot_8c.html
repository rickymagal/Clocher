Detailed Description

Replication of hot weight blobs per socket using first-touch placement.

This module provides utilities to create per-socket replicas of a contiguous "hot" blob so the inference kernel can read memory local to each socket when worker threads are socket-pinned.

Two build modes are supported: 1) Copy mode: replicate an existing source blob via memcpy(). 2) Fill mode: allocate per-socket memory, first-touch it, then let a caller-provided callback populate the replica in-place (preferred for runtime reconstruction paths to avoid cross-socket copies).

Thread pinning is delegated to the topology layer via ie_topology_bind_thread_to_socket(). This compilation unit does not implement pinning itself.
