Detailed Description

In-memory descriptor for IEBIN v1 metadata and resolved paths.

Instances are trivially copyable; no owned heap pointers are stored here. Use ie_weights_open to fill the structure, and ie_weights_close for symmetry. The binary is not kept open by this descriptor.

Deduplicated weights (lossless):

  • If a sibling model.dedup.json exists in the same directory as the opened model.ie.json, the implementation may open a dedup loader and record a small opaque handle in this descriptor.
  • Callers should treat dedup_handle as opaque and never dereference it.
  • ie_weights_close will release this handle when present.

◆ is_dedup

int ie_weights_s::is_dedup

Non-zero if this handle is backed by deduplicated artifacts.

When set, the loader found dedup metadata (e.g., model.dedup.json) and opened an internal dedup loader instance. The opaque pointer is stored in dedup_handle and must be released via ie_weights_close.

This flag is informational for callers; normal code paths can ignore it.
