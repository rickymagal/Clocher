Detailed Description

Lightweight KV cache reuse counters (process-local, single-threaded).

These counters are intended to answer one practical question: "How much previously-computed K/V data did we reuse while generating?"

The engine may call ie_kv_add_hits() with the number of cached K/V entries consumed by an attention operation, and ie_kv_add_misses() with the number of newly-produced K/V entries (typically 1 per token per layer).

The counters are process-local and not thread-safe.
